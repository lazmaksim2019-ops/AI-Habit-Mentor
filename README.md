# Neuro-Adaptive AI Habit Mentor (Backend)

Асинхронный Full-stack бэкенд для Telegram Mini App со сквозной ИИ-интеграцией, долгосрочной векторной памятью (RAG) и **полным соответствием ФЗ-152 «О персональных данных»** для развертывания в контуре РФ.

Проект спроектирован как отказоустойчивый ИИ-сервис (SaaS) для фитнес-платформ, систем изменения привычек и интерактивного менторинга.

---

## Архитектурные фичи Enterprise-уровня

### 1. Безопасность и ФЗ-152 (Compliance)
* **Полная анонимизация данных:** Архитектура полностью исключает утечку персональных данных (ПДн) пользователей во внешние LLM-сети (включая зарубежные API).
* **Кастомный маскировщик (Anonymizer Service):** Входящий поток сообщений проходит через алгоритмы фильтрации на бэкенде. Имена, фамилии, номера телефонов, email-адреса и юзернеймы автоматически маскируются в безопасные токены вида `[NAME]`, `[PHONE]`, `[EMAIL]`.
* **UUIDv4 Изоляция:** Внутри ИИ-логики и таблиц векторной памяти используется исключительно некомпилируемый случайный `UUIDv4`. Связка реального `telegram_id` с анонимным UUID хранится в изолированной локальной таблице PostgreSQL, физически размещенной на серверах в РФ.

### 2. Долгосрочная векторная память (RAG на pgvector)
* **Эмбеддинги (768d):** Каждая сессия общения и инсайты пользователя векторизуются через асинхронные вызовы модели эмбеддингов и сохраняются в базу данных.
* **Косинусный поиск:** При каждом новом запросе бэкенд выполняет высокоскоростной косинусный поиск (оператор `<=>` в расширении `pgvector`) по текущему `user_uuid`.
* **Контекстуальный RAG:** ТОП-3 релевантных воспоминания прошлых недель извлекаются и динамически подмешиваются в системный контекст ИИ. Это позволяет ментору "помнить" долгосрочную историю триггеров, срывов и побед пользователя, обеспечивая непрерывный трек адаптации.

### 3. Асинхронный стек и высокая производительность
* **Полный Async-пайплайн:** Архитектура построена на связке `FastAPI` + `asyncpg` + `SQLAlchemy 2.0 (Async engine)` + асинхронный клиент `HTTPX`.
* **Фоновые задачи (BackgroundTasks):** Операции генерации эмбеддингов текущего диалога и их запись в векторную память вынесены в фоновые процессы FastAPI. Это минимизирует время ожидания (TTFB) для интерфейса Telegram Mini App — ответ пользователю улетает мгновенно.

### 4. Изоляция ошибок (Error Isolation)
* **Принцип неблокирующего конвейера:** Все интеграционные узлы (обращения к БД, генерация векторов, вызовы LLM) изолированы независимыми блоками `try/except` с асинхронным логированием. Сбой ИИ-провайдера или сетевой таймаут не роняют приложение — система гарантированно возвращает валидный дефолтный сценарий поддержки.

### 5. Паттерн "Стратегия" (AI Provider Agnostic)
* ИИ-логика инкапсулирована за абстрактным интерфейсом `BaseAIProvider`. Текущая реализация использует API `Gemini 3.1 Flash Lite` (с ротацией прокси для стабильной работы), но архитектура позволяет бесшовно переключить весь ИИ-контур на `YandexGPT / Yandex Embeddings` изменением одной строки конфигурации в `.env`.

## Технологический стек

* **Framework:** FastAPI, Pydantic v2 (строгая валидация схем данных), Pydantic Settings (12-Factor App config)
* **Database & ORM:** PostgreSQL, `pgvector` extension, SQLAlchemy 2.0 (Asyncio style), asyncpg
* **AI Core & Network:** Gemini API (`gemini-3.1-flash-lite`, `text-embedding-004`), HTTPX (асинхронный клиент с поддержкой прокси-авторизации)
* **Security:** Regular Expressions маскирование ПДн, UUIDv4 реляционная изоляция

## Пайплайн обработки запроса (Request Flow)

```text
Telegram Client (TMA) → message + telegram_id
    │
    ▼
Найти или сгенерировать анонимный user_uuid по telegram_id (Таблица UserLink)
    │
    ▼
anonymize_text(message) → очищенный текст (маскирование ПДн)
    │
    ▼
BaseAIProvider.get_embedding() → Вектор сообщения (List[float]) через асинхронный HTTPX
    │
    ▼
get_relevant_memory() → TOP-3 воспоминания из UserVectorMemory (pgvector cosine search <=>)
    │
    ▼
SELECT FROM UserHabit → Сбор сухой статистики активных привычек и чек-листов
    │
    ▼
Сборка единого System Prompt (Инструкция + Статистика + Векторная память) → Вызов ИИ-модели
    │
    ▼
Background task: Асинхронное сохранение вопроса и ответа в векторную память (не блокирует UI)
    │
    ▼
JSON { "reply": "..." } → Отправка в Telegram Mini App
```

```
├── .env.example                # Шаблон переменных окружения

├── main.py                     # Точка входа FastAPI (Uvicorn сервер, CORS, Lifespan)

    ├── api/
    │   ├── endpoints.py        # API-эндпоинты (POST /chat, POST /habits/log, /health)

    ├── core/

    ├── database/
    │   ├── models.py           # ORM модели SQLAlchemy 2.0 (Mapped/mapped_column)
    │   ├── repository.py       # Репозиторий векторного поиска и работы с БД

            └── gemini.py       # Реализация провайдера Gemini с поддержкой прокси
```

## Быстрое развертывание (локально)

```bash
pip install -r requirements.txt
```

Настроить окружение: создать `.env` на основе `.env.example`, заполнить параметры БД и API-ключи.

Подготовить базу данных (включить pgvector):

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

Запустить сервер:

```bash
python main.py
```

Сервер на `http://localhost:8000`. Swagger: `http://localhost:8000/docs`.
