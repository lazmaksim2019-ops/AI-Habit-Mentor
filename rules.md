# Rules for AI-Agent (Vibe Coding Framework)

## 1. Концепция проекта
Разработка "Нейро-Адаптивного ИИ-Ментора привычек" (Telegram Mini App) с акцентом на Enterprise-безопасность, соответствие ФЗ-152 (РФ), анонимизацию данных и долгосрочную векторную память.

## 2. Архитектурные ограничения
* **Backend:** Python (FastAPI), полностью асинхронный стек (`async/await`), Pydantic v2 для валидации схем, HTTPX для запросов.
* **Database:** PostgreSQL + расширение `pgvector`. База хранит анонимизированные данные.
* **AI-Контур:** Модульная структура с возможностью переключения провайдеров (в данный момент используется Gemini API via HTTPX/SDK, в перспективе — Yandex Cloud SDK).
* **Локализация:** Код пишется с расчетом на деплой на серверах в РФ.

## 3. Правила обработки данных (Контур ФЗ-152)
* **Запрет на ПДн:** Никакие явные Персональные Данные (имена, фамилии, юзернеймы, телефоны, email, геопозиция) не должны попадать в ИИ-контур (Gemini/YandexGPT).
* **Middleware Анонимизации:** Весь входящий текст от пользователя перед отправкой в LLM должен проходить через функцию деперсонализации (маскирование паттернов по регулярным выражениям и замена на токены вроде `[NAME]`, `[PHONE]`).
* **Связка ID:** В базе данных существует жесткая таблица сопоставления Telegram `user_id` со случайным `UUIDv4`. В ИИ-логику передается ТОЛЬКО `UUIDv4`.

## 4. Логика Долгосрочной Памяти (PostgreSQL + pgvector)
* История диалогов и инсайты пользователя векторизуются через `text-embedding-004` (Gemini) и сохраняются в таблицу `user_memory_vectors` (поля: `id`, `user_uuid`, `embedding` (vector), `content_text`, `created_at`).
* При каждом запросе пользователя бэкенд выполняет асинхронный косинусный поиск по векторам (`<=>`) текущего UUID, достает ТОП-3 релевантных воспоминаний прошлых недель и подмешивает их в системный контекст (RAG).

## 5. Требования к качеству кода
* Строгая типизация во всем проекте (микро-типы, Pydantic-схемы).
* Изоляция ошибок (`try/except` с асинхронным логированием через встроенный `logging`). Сбой генерации эмбеддинга или памяти не должен ломать основной флоу записи привычки (принцип `continueOnFail`).
* Использование паттерна "Абстрактная фабрика" или "Стратегия" для AI-провайдеров (`BaseAIProvider`, `GeminiProvider`, `YandexCloudProvider`).
